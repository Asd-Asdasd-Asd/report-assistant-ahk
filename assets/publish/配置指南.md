# 麦旋风配置指南

本指南用于修改快捷语、触发词和可选功能。普通使用不需要修改配置。

## 配置文件

配置文件位于：

```text
%LOCALAPPDATA%\MedExReportAssistant\config.ini
```

配置只在程序启动时读取。修改前先退出麦旋风，保存后重新启动。

文件编码必须保持为带 BOM 的 **UTF-16 LE**。建议使用 Windows 记事本编辑；另存时选择 `UTF-16 LE`。不要把配置文件移动到 EXE 同目录。

## 基本结构

```ini
[Config]
SchemaVersion=1

[Features]
GlobalHjklArrows=false

[Hotstring.builtin-red]
Enabled=true
Name=红字插入
Trigger=;red
Text=（见图）
Mode=red-reset
```

`SchemaVersion` 不要手动修改。新版本可能在创建备份后补充缺失的程序默认项，但不会覆盖已有配置值。

## 可选全局方向键

```ini
[Features]
GlobalHjklArrows=true
```

启用后：

| 按键 | 作用 |
| --- | --- |
| `RAlt+H` | Left |
| `RAlt+J` | Down |
| `RAlt+K` | Up |
| `RAlt+L` | Right |

该功能默认关闭并且作用于所有应用。启用前应退出仍注册相同按键的 legacy/compatibility script。

## Hotstring section

每项 section 必须命名为：

```text
Hotstring.builtin-<stable-id>
Hotstring.custom-<user-id>
```

支持字段：

| 字段 | 含义 |
| --- | --- |
| `Enabled` | `true` 注册，`false` 禁用 |
| `Name` | 便于识别的名称 |
| `Trigger` | 用户输入的完整触发词 |
| `Text` | 插入的文字 |
| `Mode` | `text`、`red-reset` 或 `red-left4` |

未知字段会被忽略。不要添加 `Order`；section 在文件中的顺序就是优先级。

## Mode

- `text`：插入普通黑字。
- `red-reset`：正文使用黑字，末尾完整的 `（见图）` 使用红字，然后恢复后续输入颜色。
- `red-left4`：正文使用黑字，末尾 `（见图）` 使用红字，跳过颜色复位并按固定规则移动光标。

如果 red mode 的 `Text` 已经以 `（见图）` 结尾，程序会把该标记作为唯一红色内容，不会重复追加。`text` mode 不会追加红色标记。

## Text 转义

- 两个字符 `\n` 表示换行。
- 两个反斜杠 `\\` 表示一个普通反斜杠。
- 配置文字只作为文本发送，不会作为 AutoHotkey code 执行。

## 自定义示例

```ini
[Hotstring.custom-warning]
Enabled=true
Name=重点提示
Trigger=;warning
Text=请重点关注该病灶
Mode=red-reset
```

该示例插入黑色正文，并在末尾追加红色 `（见图）`。

## 重复和无效配置

- 相同 `Trigger` 出现多次时，第一个 enabled、有效且成功注册的 section 生效。
- 空 `Trigger`、未知 `Mode` 或无效 `Enabled` 会跳过该项。
- 文件缺失、不可读、schema 不支持或没有有效项时，程序使用安全的内置默认值。
- 不要配置 position、coordinate、timing、offset 或 Left-count；这些不是用户配置字段。

如果修改后行为异常，退出程序并恢复配置文件备份，或联系维护者。不要发送包含患者信息的配置示例、截图或日志。
